Feature Roadmap — Goods-Out Queue & Paging System

(Using existing LRS pagers + your own modern software)

PHASE 1 — Core MVP (2–4 weeks)

Goal: Replace the basic LRS website with something that actually works for gatehouse/day-to-day use.

✔ 1. Live Vehicle Queue

  • Add vehicle (Reg, PO/Ref, Pager #, Notes)

  • Show “elapsed time” automatically

  • Colour indicators (green/amber/red)

  • Sort by longest waiting

✔ 2. Notify (Buzz Pager)

  • Button to send page to TX-7471

  • Safe interface to prevent double paging

  • Logs “notified at” timestamp

✔ 3. Complete / Release

  • Simple “Release Vehicle” button

  • Logs time and operator

✔ 4. Operator Login (PIN or Entra ID)

  • Gatehouse user logins

  • Supervisor logins

✔ 5. Full Audit Trail

  • Every action recorded:

    • who

    • what

    • when

    • which bay

    • which vehicle

    • which pager

✔ 6. Local JSON / SQLite Storage

  • Local file or SQLite DB

  • Exportable

PHASE 2 — Enhanced Gatehouse Experience (4–8 weeks)

Goal: Make the system smooth, fast, and fully operational for all bays.

✔ 1. Multi-Bay View

  • Each bay shown separately

  • Or grouped view

✔ 2. Overdue Alerts

  • Auto-warn at 30/45/60 minutes

  • Optional auto-page at threshold

✔ 3. Supervisor Dashboard

  • Summary of all bays

  • Average wait times

  • Overdue vehicles

  • “By operator” activity

✔ 4. Offline Mode

  • Still works if internet drops

  • Syncs when online

✔ 5. Basic Search & Filter

  • Search by registration

  • Filter by operator

  • Filter by date

PHASE 3 — Reporting & Analysis (6–12 weeks)

Goal: Provide insights that LRS and WMS cannot give.

✔ 1. Daily/Weekly/Monthly Reports

  • Avg wait time per bay

  • Number of notifications

  • Operator performance

  • Release times

  • Peak congestion periods

✔ 2. CSV/Excel Export

One click → Excel-friendly report.

✔ 3. Shift-Based Metrics

  • AM / PM / Night

  • Compare shifts

  • Staffing optimisation

✔ 4. “What happened last Thursday at 15:15?” tool

  • Query any time window

  • Show exact operator actions

PHASE 4 — Hardware Integration (Optional)

Goal: Improve visibility and alerts inside the warehouse.

✔ 1. LED Wallboard Display

  • Shows bays + timers

  • Large-format display

  • Auto-updating

✔ 2. Tablet/Kiosk Mode

  • Gatehouse tablet mode

  • Warehouse floor view mode

✔ 3. Multi-Pattern Pager Support

  • Short buzz

  • Double buzz

  • Long buzz
    (useful for priority vehicles)

✔ 4. Secondary Alerts

  • Email/SMS (optional)

  • Alerts for supervisors only

PHASE 5 — Enterprise Integration (Future Upgrade)

Goal: Tie into Blue Yonder / WMS / yard operations if needed.

✔ 1. WMS Event Hooks

  • When trailer checked-in

  • When bay assignment changes

  • When load complete

✔ 2. API for Other Systems

Allow WMS/Yard/Transport team to push/pull data:

  • Add vehicle via API

  • Trigger notify event programmatically

  • Pull reporting into BI tools

✔ 3. SSO with Entra ID

Single sign-in for all staff.
